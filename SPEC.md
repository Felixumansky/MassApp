# 💪 MassApp — איפיון מערכת מעקב עלייה למסה

> אפליקציית ווב מותאמת לטלפון, בעברית (RTL), למעקב יומי אחרי קלוריות, ארוחות, מאקרו ומים.

---

## תהליך האיפיון — 5 לולאות שיפור

### 🔄 לולאה 1 — טיוטה ראשונית
- ישויות: ארוחות, מים.
- מסכים: דף בית עם רשימת ארוחות + כפתור הוספה.
- **ביקורת:** חסר! עלייה למסה זה לא רק קלוריות — חלבון הוא קריטי. אין מעקב משקל, אין יעדים יומיים, אין תמונת מצב.

### 🔄 לולאה 2 — הוספת מאקרו ויעדים
- נוספו: חלבון/פחמימה/שומן לכל ארוחה, פרופיל עם יעדים יומיים (קלוריות, חלבון, מים), מעקב משקל שבועי.
- **ביקורת:** הזנת ארוחה ידנית של כל ערך תשגע את המשתמש בטלפון. צריך קטלוג מאכלים מוכן + חישוב אוטומטי לפי כמות.

### 🔄 לולאה 3 — UX מהיר לטלפון
- נוסף: קטלוג מאכלים ישראליים מובנה (חזה עוף, אורז, קוטג', טונה...) עם ערכים ל-100 גרם — בחירה + כמות = חישוב אוטומטי.
- נוסף: כפתורי מים מהירים (כוס 250 / בקבוק 500 / 750).
- נוסף: חלוקה לסוגי ארוחה (בוקר/צהריים/ערב/ביניים).
- **ביקורת:** איך רואים התקדמות לאורך זמן? אין גרפים. אין גיימיפיקציה (האנטי-פטרן שזוהה: "No gamification").

### 🔄 לולאה 4 — התקדמות וגרפים
- נוסף: מסך התקדמות — גרף משקל, גרף קלוריות שבועי מול יעד.
- נוסף: טבעת קלוריות מונפשת בדף הבית + פסי מאקרו + אחוז השלמת יום.
- נוסף: הודעות עידוד דינמיות ("עוד 400 קלוריות ליעד! 🔥").
- **ביקורת:** טכני — תאריכים חייבים להיות לפי שעון מקומי (לא UTC). API צריך endpoint סיכום יומי אחד מהיר. מצבים ריקים (יום בלי ארוחות) צריכים הנחיה.

### 🔄 לולאה 5 — סופי ✅
- תאריכים: `YYYY-MM-DD` מקומי בצד לקוח.
- `GET /api/summary?date=` מחזיר הכול במכה אחת (ארוחות + מים + יעדים + סיכומים).
- מצבים ריקים עם CTA. מחיקת ארוחה/מים בהקשה. ללא התחברות — אפליקציה אישית.
- **האיפיון סגור.**

## ארכיטקטורה

```
MassApp  (פרונט)  React + Vite + Tailwind v4  ← אתה כאן
MassAPI  (באק)    Node.js + Express            ../MassAPI
Supabase (DB)     PostgreSQL                   ענן
```

הפרונט מדבר עם MassAPI דרך `/api/*` (proxy בפיתוח), MassAPI מדבר עם Supabase.

## מסד נתונים (Supabase)

| טבלה | תפקיד | שדות עיקריים |
|------|-------|---------------|
| `profile` | שורה אחת — יעדים | calorie_target, protein_target, carbs_target, fat_target, water_target_ml, goal_weight |
| `foods` | קטלוג מאכלים | name, calories_per_100, protein/carbs/fat_per_100, default_amount, unit |
| `meals` | יומן ארוחות | date, meal_type, food_name, amount, calories, protein, carbs, fat |
| `water_logs` | יומן מים | date, amount_ml |
| `weight_logs` | מעקב משקל | date (unique), weight |

## API (MassAPI, פורט 4000)

| Method | Route | תיאור |
|--------|-------|--------|
| GET | `/api/summary?date=` | סיכום יומי מלא: יעדים, ארוחות, מים, טוטאלים |
| GET/PUT | `/api/profile` | קריאה/עדכון יעדים |
| GET | `/api/foods?q=` | חיפוש בקטלוג |
| POST | `/api/foods` | הוספת מאכל לקטלוג |
| POST/DELETE | `/api/meals` | הוספת/מחיקת ארוחה |
| POST/DELETE | `/api/water` | הוספת/מחיקת מים |
| GET/POST | `/api/weights` | היסטוריית משקל / שקילה חדשה |
| GET | `/api/history?days=7` | קלוריות יומיות לגרף |

## מסכים (ניווט תחתון, 4 פריטים + FAB)

1. **🏠 היום** — טבעת קלוריות מונפשת, פסי מאקרו, מד מים עם כפתורים מהירים, ארוחות לפי סוג, הודעת עידוד.
2. **➕ הוספה (FAB)** — חיפוש בקטלוג / מאכל חופשי, בחירת כמות וסוג ארוחה, תצוגת ערכים חיה.
3. **📈 התקדמות** — גרף משקל, גרף קלוריות שבועי מול קו יעד, הוספת שקילה.
4. **👤 פרופיל** — עריכת יעדים, משקל יעד.

## עיצוב

- **מצב כהה** עקבי — רקע `#0B0E17` עם כתמי גרדיאנט (orbs) כתום/ורוד/סגול מונפשים.
- **Glassmorphism** — כרטיסים שקופים עם blur וגבול עדין.
- **צבעים:** ראשי גרדיאנט כתום→ורוד `#F97316→#F43F5E` (אנרגיה/קלוריות), מים `#38BDF8`, חלבון `#A78BFA`, פחמימות `#FBBF24`, שומן `#34D399`.
- **פונט:** Rubik (תמיכה מלאה בעברית), מספרים גדולים ובולטים.
- **אנימציות:** 150–300ms, טבעות מתמלאות, מספרים נספרים, respect ל-reduced-motion.
- **נגישות:** מטרות מגע ≥44px, ניגודיות 4.5:1, aria-labels בעברית.
